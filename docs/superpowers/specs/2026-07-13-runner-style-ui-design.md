# Runner 风格前端 UI 重构设计

## 状态

- 日期：2026-07-13
- 状态：已通过用户确认
- 参考站点：https://runner.now/
- 适用范围：`ui/` 下全部现有前端界面

## 背景

当前 UI 已具备完整的 Agent 工作台能力，包括首页任务输入、会话列表、任务对话、计划与工具事件、文件和工具预览、Trace、设置以及 VNC。此次重构只迁移 Runner 的视觉语言，不改变功能、总体架构、数据流或交互入口。

参考站的主要视觉特征是暖白纸张底色、炭黑高对比文字、陶土橙强调色、编辑式衬线标题、等宽状态文字、细边框、虚线与点阵纹理，以及克制的小圆角。项目不复制 Runner 的 Logo、字体文件、图片、文案或品牌素材。

## 目标

1. 让首页首屏、会话工作台和配置界面具有统一且可识别的 Runner 式编辑排版与工具界面气质。
2. 保留现有 WhiskerAgent / MoocManus 品牌名称、图标与文案。
3. 覆盖首页、侧边栏、会话详情、设置、Trace、文件和工具预览、VNC 外壳以及所有基础弹层和控件。
4. 保持当前信息密度、三栏工作台结构和全部业务行为。
5. 在桌面、平板和移动宽度下避免文字遮挡、横向溢出和布局跳动。

## 非目标

- 不修改路由、API、Hooks、Provider、状态管理、事件解析或业务数据结构。
- 不修改会话创建、消息发送、附件上传、Agent 模式、工具调用、Trace、设置保存或 VNC 控制逻辑。
- 不改变现有页面分区和主要组件层级。
- 不新增功能、文案、导航入口、设置项、主题切换或动画流程。
- 不引入新依赖，不下载或复制 Runner 的品牌资产和字体文件。

## 选定方案

采用“Runner 工作台化适配”，而非营销页的高强度复刻。保留项目作为操作型 Agent 工作台所需的紧凑信息密度，把 Runner 的特征集中到颜色、字体角色、边框、状态标识和局部点阵纹理中。

未采用以下方案：

- 高还原复古编辑风：视觉辨识度更强，但会降低设置、Trace 和长对话的扫描效率。
- 轻量换肤：改动风险最低，但不足以建立清晰的 Runner 风格。

## 视觉系统

### 色彩

- 页面背景：暖白 `#FBFBF9`。
- 内容面板：纯白或接近纯白，保持与背景的轻微层级差。
- 主文字：炭黑 `#252116`。
- 次级文字：中性灰，确保普通文本对比度不低于 4.5:1。
- 主强调色：陶土橙 `#A95638`，仅用于选中状态、进度和关键强调；该颜色在白底上的对比度约为 5.15:1。
- 边框：暖中性浅灰，保证在暖白和纯白表面上均可辨识。
- 成功、错误、链接等功能语义继续使用绿、红、蓝，不由陶土橙替代。

颜色通过全局语义变量提供，业务组件不新增零散的硬编码色值。

### 字体

- 首页问候、空状态和少量大标题使用系统中文宋体/衬线字体栈。
- 正文、表单和交互控件使用系统无衬线字体栈。
- 时间、工具状态、Trace 元数据和技术标签使用系统等宽字体栈。
- 紧凑面板内不使用大号衬线标题。
- 字号遵循稳定层级，不随视口宽度连续缩放；字距保持默认值。

### 形状与层级

- 普通面板、输入框和按钮使用 `4px` 至 `6px` 小圆角。
- 仅状态标签、模式切换和语义上需要成组的控件保留胶囊形态。
- 普通工作区不使用厚重阴影；弹窗和浮层使用克制的硬边阴影。
- 细实线用于区域边界，虚线用于执行时间轴和次级分隔。
- 点阵纹理只用于侧栏头部、首页空状态和预览标题区，不能覆盖长文本和主要交互区域。

### 图标与动效

- 继续统一使用现有 Lucide 图标，不引入 Emoji 或另一套图标语言。
- 交互反馈只改变颜色、透明度或位移，不改变布局尺寸。
- 微交互保持在约 `160ms` 至 `220ms`，并遵循 `prefers-reduced-motion`。

## 页面与组件设计

### 全局框架

维持左侧会话栏、中间对话区和可选右侧预览区。全局背景、边框、字体角色和基础控件由语义设计变量统一控制。现有侧栏宽度、预览调整宽度、固定头部和底部输入区的行为不变。

### 首页

- 问候标题使用衬线排版，第二层文字使用更低强调度。
- 输入区改为小圆角硬边命令框，保留附件、模式切换和发送按钮的原位置与行为。
- 推荐问题改为紧凑的描边条目，不做营销页式大 Hero，也不添加新文案或装饰卡片。

### 左侧会话栏

- 通过细边框、局部点阵和更明确的选中状态建立工具栏感。
- 新建任务保持原入口和快捷键，视觉上成为炭黑主操作。
- 当前会话使用白色表面、陶土橙状态标记和清晰的标题/摘要/时间层级。
- 删除菜单和折叠行为保持不变。

### 会话与执行时间线

- 用户消息使用浅中性纸张块；Agent 正文保持无框，以提升长文本阅读性。
- 步骤和工具调用使用紧凑执行记录、虚线时间轴和等宽状态标签。
- 计划面板、加载状态、空状态和错误状态保持原触发条件与文案，只统一视觉样式。
- 底部输入区仍保持固定位置和全部原有状态。

### 文件、工具与 VNC 预览

- 右侧预览从悬浮卡片改为平面工具窗口，降低圆角和阴影。
- 标题栏使用等宽状态文字和轻量点阵纹理。
- 文件正文、终端输出、浏览器画面和 VNC 远程桌面内容不变。
- VNC 只修改覆盖层、连接状态和控制按钮外观。

### 设置

- 继续使用左侧分类、右侧表单和底部操作区。
- 当前分类使用炭黑底，陶土橙作为焦点和状态强调。
- 输入框、列表、开关、Badge、子弹窗和代码配置区统一为硬边、低阴影风格。
- 所有配置字段、说明、校验和保存逻辑保持不变。

### Trace

- 使用紧凑控制台风格，指标块、Trace 列表、Span 详情和代码区强化黑白对比与等宽排版。
- 当前选中项使用炭黑或陶土橙状态标识。
- 指标、筛选、加载、错误和详情交互保持不变。

### 基础组件与浮层

按钮、输入框、文本域、菜单、Dialog、Sheet、Tooltip、Badge、Item、Switch、Toast、Kbd 和 Sidebar 统一采用相同的色彩、边框、圆角、焦点环与动效规则。图标按钮保留可访问名称和稳定命中区域。

## 响应式与可访问性

- 保留当前侧栏和右侧预览机制，不改变信息架构。
- 在 `390px`、`1024px` 和 `1440px` 宽度检查文本换行、固定区域占位和横向溢出。
- 正文文本对比度至少达到 4.5:1，大型文字和非文本控件至少达到 3:1。
- 保留可见焦点环、键盘导航、ARIA 标签和语义状态。
- 触控目标不因视觉收紧而低于现有命中范围。
- 动效遵循减少动态效果偏好。

## 数据流与错误处理

此次重构不引入新的数据流。现有 API 请求、SSE 事件、会话状态、文件状态、工具预览选择和设置表单状态均原样保留。

加载、空状态、错误和重试路径不变，只应用统一的字体、颜色、边框和间距。禁止通过隐藏错误、吞掉状态或增加默认展示分支来达成视觉效果。

## 实现约束

- 优先调整全局设计变量和现有基础 UI 组件，再处理业务组件中的局部样式。
- 只修改样式类、语义设计变量和确有必要的纯展示结构。
- 不修改 `.env` 或任何本地运行配置文件。
- 不新增依赖、功能分支或业务兜底。
- 不进行与本次视觉重构无关的重构。

## 验证与验收

实现完成后执行：

1. 前端 ESLint 检查。
2. Next.js 生产构建。
3. 现有前端相关测试。
4. 使用真实浏览器检查 `1440px`、`1024px` 和 `390px` 视口。
5. 检查首页、会话、右侧预览、设置、Trace 和 VNC 外壳的代表状态。
6. 检查文字遮挡、横向溢出、布局跳动、焦点状态、对比度和减少动态效果。

验收结果必须同时满足：

- 首屏能够明显识别 Runner 的编辑排版与工具工作台气质。
- 全部原有功能、交互入口和状态转换保持可用。
- 没有新增文本裁切、元素重叠、横向滚动或低对比度问题。
- 项目中不存在 Runner 品牌素材或未经授权的字体文件副本。
